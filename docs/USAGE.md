# WarpHost 完整使用手册

这份手册面向安装包用户。你只需要安装 WarpHost，然后根据自己的网络条件选择云端、本地 IPv6 或 SD-WAN。

## 1. 选择哪种模式

| 你的条件 | 推荐模式 |
| --- | --- |
| 有云服务器或 VPS | 云端跃迁 |
| 本机有公网 IPv6 | 本地机房 |
| 没有公网 IP，朋友之间联机 | SD-WAN |
| 想管理本地已部署容器 | 我的实例 |

## 2. 安装

下载并运行：

```text
warphost-1.0.0-setup.exe
```

安装完成后，桌面和开始菜单会出现 WarpHost 快捷方式。

安装包已内置：

- WarpHost 主程序
- EasyTier SD-WAN 核心
- 本地教程

Docker Desktop 不会被自动静默安装。如果你要用本地机房模式，请自行安装并启动 Docker Desktop。

## 3. 第一次启动检查

打开 WarpHost 后，建议按顺序确认：

1. 能否打开教程。
2. 蓝图列表是否正常加载。
3. 如果用本地模式，Docker Desktop 是否已运行。
4. 如果用 SD-WAN，网络名称和密码是否已准备好。
5. 如果用云端模式，服务器 IP、密码、安全组端口是否准备好。

## 4. 云端跃迁

适合有 VPS 或云服务器的用户。

步骤：

1. 进入 **云端跃迁**。
2. 输入服务器 IP。
3. 输入 root 密码。
4. 选择游戏蓝图。
5. 点击 **一键跃迁**。
6. 查看日志中的连接地址。

你需要额外确认云厂商安全组已经放行游戏端口。例如 Minecraft 通常是 `25565`。

## 5. 本地机房

适合本机有公网 IPv6 的用户。

步骤：

1. 启动 Docker Desktop。
2. 进入 **本地机房**。
3. 选择游戏蓝图。
4. 点击本地部署。
5. WarpHost 会扫描公网 IPv6，尝试打开 Windows 防火墙端口，并启动容器。

成功后日志会显示类似：

```text
[240e:xxxx:xxxx::1]:25565
```

把这个地址发给支持 IPv6 的玩家即可。

## 6. SD-WAN

适合无公网 IP 或 NAT 环境。

步骤：

1. 进入 **本地机房**。
2. 在 SD-WAN 区域输入网络名称。
3. 输入至少 8 位网络密钥。
4. 点击连接。
5. 等待虚拟 IP 出现。
6. 玩家使用相同网络名称和密钥加入。

开服者把虚拟 IP 和端口发给其他玩家：

```text
10.x.x.x:25565
```

## 7. 我的实例

进入 **我的实例** 可以管理本地 Docker 容器：

- Start：启动容器
- Stop：停止容器
- Destroy：删除容器
- Refresh：刷新列表

注意：Destroy 是强制删除，本地容器数据可能无法恢复。

## 8. 蓝图说明

蓝图决定启动什么游戏服务。

示例：

```json
{
  "name": "minecraft",
  "image": "itzg/minecraft-server:latest",
  "hostPort": 25565,
  "containerPort": 25565,
  "volumeMapping": "/opt/warphost/minecraft:/data",
  "envVars": {
    "EULA": "TRUE",
    "MEMORY": "2G"
  }
}
```

字段说明：

| 字段 | 含义 |
| --- | --- |
| `name` | 蓝图名称，也是容器名的一部分 |
| `image` | Docker 镜像 |
| `hostPort` | 宿主机开放端口 |
| `containerPort` | 容器内部端口 |
| `volumeMapping` | 数据卷映射 |
| `envVars` | 容器环境变量 |

## 9. 排错速查

| 现象 | 优先检查 |
| --- | --- |
| SSH 失败 | IP、root 密码、22 端口、安全组 |
| 云端部署后连不上 | 云安全组、系统防火墙、游戏端口 |
| 本地部署失败 | Docker Desktop 是否启动 |
| 未检测到 IPv6 | 宽带、路由器、系统网络配置 |
| 防火墙规则失败 | 是否以管理员身份运行 |
| SD-WAN 启动失败 | EasyTier 是否随安装包分发、网络名和密码是否合法 |
| 没有虚拟 IP | 防火墙、网络阻断、其他玩家是否同网同密钥 |
| 实例列表为空 | Docker 是否运行，是否存在 `warphost-` 容器 |

## 10. 推荐流程

新手推荐：

1. 先用 SD-WAN 连接确认网络可用。
2. 再用本地机房或云端跃迁启动游戏。
3. 最后进入我的实例确认容器状态。

服主推荐：

1. 固定蓝图。
2. 固定端口。
3. 固定网络名称。
4. 将连接地址和教程发给玩家。
